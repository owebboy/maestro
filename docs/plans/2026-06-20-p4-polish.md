# P4 — Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the pluggable-backend system production-ready: harden graceful degradation for thin backends and missing transports, ship an automated validation suite wired into hooks + `AGENTS.md`, document every adapter plus a "write-your-own-adapter" guide, bring Codex `agents/openai.yaml` + manifests to full parity, and bump the version for the breaking change.

**Architecture:** Three kinds of work. (1) **Degradation** is adapter-internal: each profile's degradable/optional ops declare a concrete fallback, and the contract specifies a single graceful "no transport / unsupported op" failure behavior — no skill edits. (2) **Validation** is a new `bin/validate-maestro` (Python 3 stdlib) that statically checks the whole package (contract complete, every adapter shape-complete + covers 12 ops, skills stay abstract, config schema valid, item records valid) with a unittest suite; it joins the `AGENTS.md` validation block and a PostToolUse hook. (3) **Docs + parity** are content sweeps over `docs/`, every `agents/openai.yaml`, and the four manifests.

**Tech Stack:** Python 3 stdlib (validator + tests), Bash (hooks), JSON (manifests/config), Markdown (docs, profiles, skills, openai.yaml).

- **Depends on P0–P3** (all must be complete; this plan validates and documents what they built). Cite P0 Locked Decisions, P2 `_shape.md`, and P3 native-state rules rather than re-deriving.
- **Degradation never silently downgrades correctness.** A degradable op uses its documented fallback; an unsupported *required-for-the-current-item* op (e.g. `set_subtasks` for a tracked item on a backend that has neither sub-issues nor task-lists) must surface a clear message, not silently drop progress (spec §"No silent caps" principle).

```markdown
## Degradation

Capability flags (profile header `supports`) drive behavior. Rules:

1. Degradable required ops:
   - link_artifact: if no native artifact field -> append `- <kind>: <ref>` under `## Artifacts` in the body.
   - comment: if no native comments -> append the text under `## Notes` in the body.
2. Optional ops:
   - capture_raw: captureMode=backend needs create_item; if the backend can't, use local .maestro/inbox.md.
   - search: if no native search -> list_items + case-insensitive local match.
   - relate: if no native relations -> a comment ("duplicate-of #X") + set_status(duplicate) for duplicates.
3. set_subtasks/set_subtask_state for a TRACKED item:
   - prefer native sub-issues (supports: subissues); else a `- [ ]` task-list in the body (supports:
     subtasks-as-tasklist); if NEITHER is possible, STOP and tell the user this backend cannot track
     plan progress natively — offer to keep the item `light` (steps live only in .maestro/work/<id>/plan.md).
4. No transport available (detection yields none): STOP. Print which of MCP/CLI/API to configure and the
   one-line auth command for each. NEVER fall back to the files adapter silently.
5. Unmapped native status (linear/jira read path): report `inbox` + warn; prompt to add to config.statusMap.
```

- [ ] **Step 3: Implement the validator**

Create `bin/validate-maestro`:
```python
#!/usr/bin/env python3
"""Static validation of the Maestro package: contract, adapters, skill abstraction."""
import pathlib
import re
import sys

OPS = ["create_item", "get_item", "update_item", "set_status", "list_items",
       "set_subtasks", "set_subtask_state", "link_artifact", "comment",
       "capture_raw", "search", "relate"]
STATUSES = ["inbox", "triaged", "reviewed", "planned", "in-progress", "in-review",
            "done", "wont-fix", "deferred", "duplicate"]
SHAPE_SECTIONS = ["## Capabilities", "## Transport detection", "## Status mapping",
                  "## Field mapping", "## Operations"]
LEGACY = ["conductor/", "issues/", "metadata.json", "tracks.md"]
BACKEND_TOKENS = ["mcp__gitea", "gh issue", "glab issue", "mcp__linear",
                  "mcp__atlassian", "jira issue"]
LIFECYCLE = ["triage", "issue-review", "issue-advance", "new-track", "implement",
             "issue-close", "status", "manage", "session-wrap-up", "workflow-router",
             "uat-create", "uat-run", "codebase-review"]


def _read(p):
    return p.read_text() if p.exists() else ""


def check_contract(root):
    txt = _read(pathlib.Path(root) / "assets/maestro/CONTRACT.md")
    errs = []
    for op in OPS:
        if op not in txt:
            errs.append(f"CONTRACT missing op {op}")
    for st in STATUSES:
        if st not in txt:
            errs.append(f"CONTRACT missing status {st}")
    for sec in ("## Degradation", "## Config keys"):
        if sec not in txt:
            errs.append(f"CONTRACT missing section {sec}")
    return errs


def check_adapters(root):
    base = pathlib.Path(root) / "assets/maestro/adapters"
    errs = []
    for f in sorted(base.glob("*.md")):
        if f.name.startswith("_"):
            continue
        txt = f.read_text()
        # files adapter uses a simpler shape; forge/native adapters use the full shape
        if f.stem != "files":
            for sec in SHAPE_SECTIONS:
                if sec not in txt:
                    errs.append(f"{f.name} missing section {sec}")
        for op in OPS:
            if op not in txt:
                errs.append(f"{f.name} missing op {op}")
    return errs


def check_skills_abstract(root):
    base = pathlib.Path(root) / "skills"
    errs = []
    for name in LIFECYCLE:
        txt = _read(base / name / "SKILL.md")
        for bad in LEGACY + BACKEND_TOKENS:
            # allow the bare word "issues" in prose; only flag path-like "issues/"
            if bad in txt:
                errs.append(f"skills/{name} references forbidden token {bad!r}")
    return errs


def validate(root):
    return check_contract(root) + check_adapters(root) + check_skills_abstract(root)


def main(argv=None):
    root = (argv or sys.argv[1:] or ["."])[0]
    errs = validate(root)
    if errs:
        print("MAESTRO VALIDATION FAILED:")
        for e in errs:
            print(f"  - {e}")
        return 1
    print("maestro validation ok")
    return 0


if __name__ == "__main__":
    raise SystemExit(main())
```
